# 対応

次のコマンドを実行するだけです。

```bash
git remote prune origin
```
# 終わりに

この事象はリモートブランチが何らかの状態で削除されているが、ローカルリポジトリに残ってしまっているときに発生してしまうことが多いようです。
  
しかし、今回の私の事象に関しては、```master```ブランチが残っている状態だったので、普通に```GitHub Codespaces```の環境が壊れていたようです。
